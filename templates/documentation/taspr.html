Temperature and Precipitation Data

These endpoints provide access to historical and projected downscaled temperature and precipitation data at a resolution of 2km. Temperature and precipitation data can be accessed individually or together. Historical data were derived using the CRU TS 4.0 dataset. Projections were derived from MRI-CGCM3 and NCAR-CCSM4 model outputs, as well as a 5-model average, under the RCP 4.5, RCP 6.0, and RCP 8.5 emissions scenarios. Data can be accessed for individual years, summarized means across decades or multi-decade eras, or minimum, mean, and maximum summaries for January or July across 1901–2100.

Links to academic references and source datasets are included at the bottom of this page.

Service endpoints

Seasonal averages (point query)

Query SNAP's 2km downscaled temperature or precipitation data for a single point specified by latitude and longitude.

Endpoint Example URL
Temperature point query /temperature/point/65.0628/-146.1627
Precipitation point query /precipitation/point/65.0628/-146.1627
Temperature and precipitation point query /taspr/point/65.0628/-146.1627
CSV output is also available by appending ?format=csv to the URL.

Seasonal averages (area query)

Query SNAP's 2km downscaled temperature or precipitation data for a specific area of interest polygon ID and aggregate by mean.

Endpoint Example URL
Temperature area query /temperature/area/19010208
Precipitation area query /precipitation/area/19010208
Temperature and precipitation area query /taspr/area/19010208
CSV output is also available by appending ?format=csv to the URL.

Precipitation frequency (point query)

The following query will return precipitation frequency for various return intervals and durations across different models and eras.

Endpoint Example URL
Precipitation frequency /precipitation/frequency/point/65.028/-146.1627
CSV output is also available by appending ?format=csv to the URL.

Yearly values (point query)

Query temperature and precipitation for each year summarized across all models and scenarios.

Endpoint Example URL
Annual mean temperature /temperature/65.0628/-146.1627
Annual total precipitation /precipitation/65.0628/-146.1627
January temperature min/mean/max /temperature/jan/65.0628/-146.1627
July temperature min/mean/max /temperature/july/65.0628/-146.1627
Min/mean/max summaries of historical and projected years are available by appending ?summarize=mmm to the URL.
CSV output is also available by appending ?format=csv to the URL.

Yearly values within provided year range (point query)

Query temperature and precipitation for each year within a provided range summarized across all models and scenarios. Valid year ranges are between 1901–2015 for historical data and 2007–2100 for projected data.

Endpoint Example URL
Annual mean temperature within year range /temperature/65.0628/-146.1627/1940/2060
Annual total precipitation within year range /precipitation/65.0628/-146.1627/1940/2060
January temperature min/mean/max within year range /temperature/jan/65.0628/-146.1627/1940/2060
July temperature min/mean/max within year range /temperature/july/65.0628/-146.1627/1940/2060
Min/mean/max summaries of historical and projected years are available by appending ?summarize=mmm to the URL.
CSV output is also available by appending ?format=csv to the URL.

              "q1": 30,
              "q3": 54
          },
          "tas": {
            "hi_std": -16.7,
            "lo_std": -23.8,
            "max": -12.9,
            "mean": -20.3,
            "median": -20.4,
            "min": -31.2,
            "q1": -22,
            "q3": -17.7
          }
        },
        ...
      },
      ...
    },
    ...
  },
  ...
}

The above output is structured like this:

{
  <period>: {
    <season>:{
      <model>: {
        <scenario>: {
          <varname>: {
            <hi_std>: <mean + standard deviation>,
            <lo_std>: <mean - standard deviation>,
            <max>: <maximum>,
            <q3>: <3rd quartile>,
            <median>: <median>,
            <mean>: <mean>,
            <q1>: <1st quartile>,
            <min>: <minimum>
          },
          ...
        },
        ...
      },
      ...
    },
    ...
  },
  ...
}

Note: The full set of statistics is only available for historical output. Projected output is limited to just the mean statistic.

Min/mean/max summaries across multiple years

Results from temperature min/mean/max queries (using the ?summarize=mmm URL parameter) will look like this:

{
  "historical": {
    "tasmax": -1.8,
    "tasmean": -4.4,
    "tasmin": -7.2
  },
  "projected": {
    "tasmax": 6.6,
    "tasmean": -1,
    "tasmin": -6.2
  }
}

Results from precipitation min/mean/max queries (using the ?summarize=mmm URL parameter) will look like this:

{
  "historical": {
    "prmax": 640,
    "prmean": 398,
    "prmin": 260
  },
  "projected": {
    "prmax": 919,
    "prmean": 499,
    "prmin": 291
  }
}

The above output for both temperature and precipitation is structured like this:

{
  <historial or projected>: {
    <temperature (tas) or precipitation (pr) max>: <max value across all models, scenarios, and years>,
    <temperature (tas) or precipitation (pr) mean>: <mean value across all models, scenarios, and years>,
    <temperature (tas) or precipitation (pr) min>: <min value across all models, scenarios, and years>
  },
  ...
}
